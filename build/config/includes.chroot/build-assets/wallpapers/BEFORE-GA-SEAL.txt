Omahon seal — use a FRESH image right before the GA pack
==========================================================
If you do not like the current `raw/omahon-seal-source.png`, do **not** ship it on a
“final” ISO. The workflow is:

1) **Generate** your final seal in your tool of choice (design app, pro model, etc.):
   - Export a **high-resolution** master PNG. **Minimum short edge 2048 px** (the
     apply script enforces this for GA). **Ideal: 4096×4096** (or larger square) so
     the 7680×4320 8K canvas stays sharp. 768 or 1024 px masters are **too small** —
     they will look soft/blown up; the pipeline does not “AI upscale” a tiny source.
   - The build places the seal centered on a deep navy canvas (`#0a0a14`).
2) **The same day** (or immediately before) you start `build-unified.sh ga`, run:

     bash build-assets/wallpapers/scripts/apply-new-omahon-source.sh /path/to/final.png

   That backs up the old raw, copies your file to `raw/omahon-seal-source.png`, and
   regenerates `8k/`, `4k/`, `1080p/` via `prepare-omahon-seal-for-build-assets.sh`.
3) **Update** `OMAHON-SEAL-PACK-READY.txt` with the new sha256 lines the script prints.
4) **Then** run the unified GA build. Hooks `0100` and `0285` will pick up the new art.

**Paid API (OpenAI, key in vault) — not free**
  python3 build-assets/wallpapers/scripts/generate-omahon-seal-openai.py --apply

  - Reads `~/.vault/keys/openai.key` or `OPENAI_API_KEY` (you are billed by OpenAI).
  - Default model `gpt-image-1.5`, `quality=high`, then Lanczos upscale to 4096 px
    short edge so 8K stays usable. Override: `OMAHON_SEAL_API_MODEL`, `OMAHON_SEAL_PROMPT`, etc.
  - `--apply` runs `apply-new-omahon-source.sh` on the result (raw + 8k/4k/1080p pack).

The seal is **not** generated inside this repo without your API key. You own billing
and pixels; the scripts only size and place them. Omahon.
